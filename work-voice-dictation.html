The problem

I write a lot — emails, code comments, posts, this site. Typing is the bottleneck.

Off-the-shelf dictation tools all required me to break flow — open a separate app, dictate, copy, paste. By the time I'd done that I could have typed it.

I wanted: hit a hotkey, speak, release, have the text appear in whatever app I'm in. Like a fast assistant typing for me.

What I built

A background Windows app:

1. Listen. A tray app holds a global hotkey — press-and-hold to record, release to transcribe.

2. Transcribe. A local Whisper model (small for speed, swap to medium for accuracy). All on-device — no API cost, no internet required.

3. Paste. Win32 AttachThreadInput + simulated Ctrl+V drops the text into whatever window has focus — VS Code, browsers, Slack, Electron apps.

4. Learn. A speech profile (SQLite) tracks corrections. After I fix the same word three times, it starts applying the correction automatically.

5. Show feedback. The tray icon pulses while recording; a silence-countdown badge shows when it'll cut off; a confidence-bordered preview lets me review low-confidence transcriptions before pasting.

The result

Faster than typing for messages over ~30 words. 46 unit tests covering hotkey state, paste logic and profile learning. Free — local Whisper, no API cost, fully offline. Personal — but the pattern generalises: this is exactly the kind of small productivity tool I'd build for a client with a repetitive transcription or data-entry task.
