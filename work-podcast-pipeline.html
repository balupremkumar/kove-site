The problem

Post-production for a one-hour podcast episode routinely takes 3–5 hours. Someone has to listen back, write show notes, pull timestamps, research guests, identify the best clips, and format everything for the feed. It's skilled work, but most of it is just patient attention. The kind AI handles well.

For a NZ health sector client, there was an added constraint: audio content included clinically sensitive material. Any cloud AI processing was off the table. The tool had to run locally and meet the Health Information Privacy Code (HIPC).

What I built

A FastAPI backend with a 7-stage pipeline that turns a raw audio file into a complete, publish-ready package, all triggered from a single upload.

The seven stages

1. Transcription: faster-whisper with VAD and word-level timestamps, producing a clean, time-coded transcript. 2. Speaker diarisation: pyannote.audio identifies and labels each speaker across the full episode. 3. Guest research: guest names extracted from the transcript, cross-referenced against public sources, bio drafted automatically. 4. Show notes: Claude writes the full package: episode title, description, themes, notable moments with timestamps, key quotes. 5. Chapter markers: Claude detects natural topic shifts, names each chapter, and outputs exact timecodes ready for your podcast host. 6. Cold open: a punchy 30-second hook written by Claude and synthesised in the host's own cloned voice via Chatterbox TTS. 7. Quality score: four dimensions rated (information density, pacing, clarity, memorable moments) with weak-segment flags so you know exactly what to tighten before publishing.

Dual backend for compliance

The pipeline has two modes controlled by a single config toggle. Cloud mode uses Claude Sonnet via the Anthropic API for fastest results. Local mode runs Qwen2.5-14B via LM Studio for fully air-gapped processing. No audio or transcript leaves the machine. The NZ health client runs local mode. The output format is identical either way.

The result

~4 hours of manual post-production reduced to under 10 minutes of pipeline time plus a quick review pass. Every episode now ships with consistent show notes, accurate timestamps, and a research brief on the guest. Work that used to get skipped when time was tight. The local backend option satisfied the client's HIPC obligations without any special procurement or legal review.
